PresentAgent

Document-to-Presentation Video Generation — 端到端文档到演示视频生成系统

论文标题
PresentAgent: A Multi-modal Framework for Document-to-Presentation Video Generation
发表会议
EMNLP 2025 System Demo
核心任务
Document → Presentation Video
技术路线
多模态 LLM 驱动的 4-Stage Pipeline
Architecture

四阶段生成流程

1

Document Processing

解析 PDF 文档，提取文本、图表、布局信息，构建结构化文档表示

2

Slide Planning

基于 LLM 规划幻灯片结构与内容分配，确定每页的主题、要点和视觉元素

3

Narration & Audio

为每张幻灯片生成解说文本，通过 TTS 合成同步旁白音频

4

Video Assembly

将幻灯片画面与音频时间轴对齐，合成最终演示视频

生成管线详解（Step 1–4）

1 Outline Generation 文档解析与大纲生成

输入文档（PDF、网页、Markdown、博客等）首先经过结构化解析，将内容映射为层级化的 topic–subtopic 树形结构。随后 LLM 基于文档结构生成演示大纲（Outline），将文档分割为若干语义块，每个语义块对应一张计划中的幻灯片。

2 Template Selection 模板检索与匹配

为每个语义块分配一个幻灯片类型（标题页、项目列表、图表说明等），并通过检索匹配找到最合适的模板布局。匹配依据包括内容语义和结构线索（非简单关键词匹配）。选定模板后，系统定义一组可编辑操作（replace_text、insert_image、add_list）用于后续内容填充。

3 Slides & Notes Generation 幻灯片与旁白生成

这是核心生成步骤，由VLM（视觉语言模型）驱动。基于大纲和选定的模板，模型同时完成两项任务：
(a) 幻灯片内容填充——将论文中的关键信息（标题、要点、图表）映射到模板布局中，从论文中检索并插入相关图片；
(b) 旁白脚本生成——LLM 将每张幻灯片的关键信息改写为自然口语化叙述，避免密集文本和技术术语，时长控制在 30–150 秒。
最终输出：一套完整的幻灯片 + 每张幻灯片对应的旁白文本。旁白文本随后通过 MegaTTS 3 合成为 24kHz 高保真音频。

4 Video Generation 视频合成

使用 FFmpeg 将幻灯片图像与旁白音频进行时间对齐合成。每张幻灯片显示时长 = 其对应旁白音频的时长。支持淡入/淡出转场效果和可选字幕叠加。最终输出 1080p 标准 .mp4 视频文件。

评估管线详解（Step 5–6）

5 Prompt Design 评估提示词设计

为每个评估维度设计专用 Prompt。客观评估：为每篇文档人工编制 5 道多选题（覆盖主题识别、结构理解、核心论点提取）。主观评估：设计 3 维度评分 Prompt（叙事连贯性、视觉/音频吸引力、理解难度），引导 VLM 从观众视角打分。

6 Model Evaluation VLM 驱动评分

客观测验：使用 Qwen-VL-2.5-3B 观看完整视频后回答 5 道题，正确数即为理解得分（0–5）。主观评分：视频部分用 Qwen-VL 评估（内容/视觉/理解），音频部分用 Qwen-Omni-7B 评估（内容/音频/理解），均为 1–5 分。由于长视频超过 2 分钟时单一多模态模型难以联合处理，采用分段评估策略。

PresentEval 评估框架

双路径评估方案：客观路径通过自动生成 quiz 衡量信息保留度，主观路径从视频质量、音频质量等维度打分。

局限性

  • 依赖闭源商业 LLM API（Claude / GPT），成本高且不可控
  • Quiz-based 评估仅覆盖信息保留维度，无法衡量表达策略质量
  • 缺乏个性化定制能力，无法适配不同演讲风格或听众背景
  • 视频输出为终端形态，不可二次编辑，实用性受限

SlideTailor

Preference-Guided Paper-to-Slides Generation — 基于偏好蒸馏的论文到幻灯片个性化生成

论文标题
SlideTailor: Preference-Guided Paper-to-Slides Generation via Chain-of-Speech
发表会议
AAAI 2026 主会
研究机构
National University of Singapore (NUS)
核心任务
Paper → Editable .pptx Slides
Input Design

用户输入设计：示例驱动的偏好表达

用户不需要编写复杂的指令，而是通过提供一对示例幻灯片（sample pair）和一个模板（template）来表达自己的偏好。系统从示例对中隐式学习用户的选择偏好（内容筛选、组织方式、详略取舍），并按模板风格生成最终幻灯片。这种 "Show, don't tell" 的范式极大降低了用户的使用门槛。

Architecture

三阶段生成框架

1

Implicit Preference Distillation

从用户提供的示例对中，通过对比学习提取隐式的选择偏好，构建用户偏好模型

2

Preference-Guided Planning

利用蒸馏得到的偏好模型指导幻灯片的内容规划与结构设计，确保输出符合用户风格

3

Slide Realization

根据规划结果和模板约束，生成可编辑的 .pptx 幻灯片文件

三阶段框架详解

1 Implicit Preference Distillation 隐式偏好蒸馏

从用户的两种隐式输入中蒸馏出显式的、结构化的偏好表示 P = PC ∪ PA。这一阶段是整个框架的基础，将无标签的用户信号转化为可执行的生成约束。

内容偏好 PC

输入：论文-幻灯片样本对 (Dref, Sref)
方法：LLM 将样本对中的转换建模为隐函数 f: Dref → Sref，推断用户的内容选择策略、强调/省略模式、排序偏好
输出：结构化偏好档案，包含叙事流程（如 Title → Background → Method → Results → Future Work）和章节级格式偏好

审美偏好 PA

输入：用户提供的 .pptx 模板文件
方法：VLM 推断模板中每个幻灯片的功能角色（标题区、正文区、图片区），同时从原始 .pptx 解析精确元数据（边界框位置、尺寸）
输出：结构化审美模式，描述每个模板 slide 的内容 Schema（位置、尺寸、类型等）

2 Preference-Guided Slide Planning 偏好引导的内容规划

三个 LLM-powered Agent 协同工作，将蒸馏出的偏好 P 转化为具体的幻灯片规划方案：

Paper Reorganizer（论文重组器）

根据内容偏好 PC 对输入论文进行面向演示的重构——调整叙事流程（如先讲动机再讲方法还是反过来）、控制各部分详略（如精简 Related Work、展开 Results），输出一份"面向演示的文档"。这一步不是简单的摘要，而是结构化的内容重组。

Slide Outline Designer + Chain-of-Speech

将重组后的内容分割为连贯的幻灯片大纲，每张幻灯片明确其"要传达的信息"和"视觉提示"（如需要插入论文中的哪张图表）。

核心机制——Chain-of-Speech：在规划每张幻灯片的同时，同步模拟演讲者的口头叙述。例如规划"实验结果"幻灯片时，CoS 会生成类似"Our experiments demonstrate that..."的演讲脚本。这一机制确保视觉内容与预期口头表达深度对齐，而非先做幻灯片再补旁白。输出的演讲脚本 T 还可用于下游 TTS 生成个性化语音。

Template Selector（模板选择器）

基于审美偏好 PA，为每张幻灯片匹配用户模板中最合适的布局。例如标题页选用 slide-0（标题+副标题），结果页选用 slide-2（图文混排+表格）。每张幻灯片附带 Layout Recommendation 和 Layout Justification。

3 Slide Realization 幻灯片实现

将规划方案落地为实际的 .pptx 文件。布局感知 Agent将规划内容（标题、文本、图表）映射到选定模板的具体元素（文本框、图片占位符）——可能涉及修改、替换或插入操作。随后代码 Agent生成可执行的 Python 代码，通过 python-pptx 直接编辑 .pptx 文件。这一编辑式策略保留了模板的原始布局和主题，输出完全可编辑的幻灯片，用户可进一步微调。

Key Innovation

Chain-of-Speech 机制

SlideTailor 的核心创新之一。通过构建 "Chain-of-Speech"（语音链），系统在规划阶段模拟人类演讲者的思维过程：先确定要"说"什么（口语化内容规划），再决定如何"展示"（视觉布局与排版）。这一机制使得生成的幻灯片在内容组织上更符合口头演讲的逻辑流，而非简单的论文段落搬运。Chain-of-Speech 将"演讲意图"显式建模为中间表征，桥接了论文内容与幻灯片表达之间的语义鸿沟。

PresentAgent 方向分析

优势

  • ✓端到端方案完整度高，从文档到视频一气呵成，用户体验流畅
  • ✓多模态融合能力强，文本、图像、音频协同生成
  • ✓PresentEval 评估框架设计有创新性，Quiz + Subjective 双路径
  • ✓GPT-4o-Mini 在部分指标上已超越人类基线，技术可行性得到验证
  • ✓产品形态清晰，可直接面向非技术用户
  • ✓输入门槛低，仅需 PDF 文档，零配置使用

劣势

  • ✗严重依赖闭源商业 LLM（Claude/GPT），成本与可控性堪忧
  • ✗缺乏个性化机制，无法适配不同演讲者风格或听众需求
  • ✗视频输出不可编辑，实际使用场景受限（无法微调）
  • ✗评估维度单一，Quiz Accuracy 难以反映演示质量的全面性
  • ✗仅发表在 System Demo 赛道，方法论深度和学术贡献度有限
  • ✗Pipeline 各模块串行耦合，灵活性和可扩展性不足
  • ✗缺乏消融实验，各模块贡献度无法量化评估

SlideTailor 方向分析

优势

  • ✓Chain-of-Speech 机制创新性强，模拟人类演讲思维链
  • ✓偏好蒸馏实现个性化生成，用户满意度更高
  • ✓示例驱动的偏好表达方式自然直观，降低使用门槛
  • ✓输出可编辑的 .pptx 文件，实用性远高于视频终端
  • ✓PSP 数据集构建工作扎实，100K 组合规模可观
  • ✓6 维度评估体系全面，消融实验设计严谨
  • ✓AAAI 主会发表，学术贡献获得顶级会议认可
  • ✓框架设计模块化，各组件可独立替换和升级

劣势

  • ✗需要用户提供示例对和模板，初始输入成本高于 PresentAgent
  • ✗偏好蒸馏的冷启动问题未充分讨论
  • ✗仅覆盖学术论文领域，泛化能力存疑
  • ✗模板系统灵活性有限，自定义扩展空间不足
  • ✗未涉及音频/视频多模态，输出形态相对单一
  • ✗LLM 后端的 API 成本问题同样存在

Q&A

关键问题讨论

核心问题的思考与解答

Q1
这两篇论文的核心差异是什么？如果只能选一个方向深入研究，你选哪个？
核心差异在于任务定义和创新维度：PresentAgent 聚焦"文档→视频"的端到端生成，创新点在于多模态融合与 Pipeline 组装；SlideTailor 聚焦"论文→幻灯片"的个性化生成，创新点在于偏好蒸馏与 Chain-of-Speech。

如果只选一个，我会选 SlideTailor 方向，原因是：（1）AAAI 主会的方法论深度更高，研究空间更大；（2）个性化/偏好学习是学术界的热点话题，延展性强；（3）Chain-of-Speech 机制有独立的研究价值，可以深入挖掘。但最终的理想方案是两者的融合。
Q2
SlideTailor 的 Chain-of-Speech 机制，和你之前了解的 CoT（Chain-of-Thought）有什么异同？
相同点：都是通过构建中间推理链路来提升生成质量，核心思想是"分步思考优于一步到位"。

不同点：CoT 主要用于 LLM 的文本推理任务（数学、逻辑），中间步骤是文本 token；而 CoS 的中间步骤是模拟口头演讲的语义单元，它的推理链路是从"论文内容"到"演讲意图"再到"幻灯片布局"的跨模态映射。CoS 本质上是一种模态转换的推理桥，将学术写作的语义空间映射到口头表达的语义空间。
Q3
这两篇论文都依赖 LLM API，如果开源模型持续进步，这个方向还有独立的研究价值吗？
有，而且会越来越有价值。原因有三：

（1）LLM 是基础设施而非研究终点。就像计算机视觉研究不因相机进步而消失，PPT 生成的核心问题（内容筛选、结构规划、视觉表达、个性化适配）是独立于 LLM 的研究课题。

（2）SlideTailor 已经验证了这一点——其核心贡献（偏好蒸馏、CoS）是框架层面的创新，与具体使用哪个 LLM 无关。实验也表明切换后端（Qwen → GPT-4.1）可以带来性能提升。

（3）随着开源模型逼近闭源模型能力，成本和可控性问题将逐步解决，届时研究重心会从"能不能做"转向"如何做得更好"，这正是学术研究的价值所在。
Q4
你提到的"融合方案"具体怎么做？有哪些技术挑战？
融合方案的核心思路是：以 SlideTailor 的偏好驱动框架为骨架，注入 PresentAgent 的多模态输出能力。

具体架构：用户输入示例对 + 模板 → 偏好蒸馏 → Chain-of-Speech 规划（生成演讲脚本）→ 幻灯片生成（.pptx，可编辑中间产物）→ TTS 合成旁白 → 视频组装（最终产物）。

技术挑战：（1）CoS 生成的演讲脚本如何与 TTS 时间轴精确对齐；（2）偏好蒸馏在视频生成场景下是否依然有效（视频比幻灯片多一个时间维度）；（3）输出两种格式（.pptx + .mp4）的联合评估标准如何设计。这些挑战本身也是研究贡献点。

On SlideTailor

SlideTailor 方向优化

领域泛化

从学术论文扩展到技术文档、商业报告、课堂笔记。核心挑战是不同文档类型结构差异大，偏好蒸馏能否跨领域迁移。

目标会议：ACL / AAAI

偏好建模深化

多样本聚合提取更鲁棒的用户画像，交互式偏好学习（Active Learning），偏好维度细分。

目标会议：EMNLP

Chain-of-Speech 深化

听众感知（专家 vs 本科生生成不同脚本）、多语言跨语言演讲规划、时间约束感知（5 分钟 vs 20 分钟）、情感风格控制。

目标会议：ACL / NAACL

端到端训练

替代 Prompt-based 方案，构建标注数据集训练专用 Slide Generation Model（基于 LLM 做 SFT），降低成本提升可控性。

目标会议：ACL / EMNLP 主会
On PresentAgent

PresentAgent 方向优化

个性化视频生成

引入偏好条件（节奏 / 信息密度 / 正式度），声纹个性化（几秒语音克隆），演讲风格迁移。

目标会议：ACM Multimedia

动态幻灯片与动画

元素级动画（文字逐条出现、图表渐进显示），智能转场设计，镜头感（放大关键数据区域）。用 reveal.js / animate.css 替代静态渲染。

目标会议：CHI

评估体系升级

扩大评估到 50–100 篇，跨模态对齐评估（不只分别评视频和音频），人类评估标准化（MOS + A/B Test），构建标准 Benchmark。

目标会议：ACL
Fusion (Recommended)

融合创新 ✦

Preference-Guided Doc-to-Video

以 SlideTailor 偏好框架为骨架 + PresentAgent 多模态管线为输出。Pipeline：偏好蒸馏 → CoS 规划 → 幻灯片生成（.pptx）→ TTS 旁白 → 视频组装（.mp4）。双输出格式。

技术挑战：偏好从静态到动态的传递、CoS 脚本与 TTS 时间对齐、联合评估标准设计。

目标会议：AAAI / ACL / CVPR 主会

Interactive Presentation Agent

多轮交互打磨演示：生成初版 → 用户反馈 → 迭代优化。Human-in-the-Loop 多模态生成框架。

目标会议：NeurIPS / ICML
